# Contribution Guidelines

Welcome to our project! We're excited to have you contribute. Please follow these guidelines to ensure a smooth collaboration.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a feature branch for your work
4. Make your changes with clear, descriptive commit messages
5. Push to your fork and submit a pull request

## Code Standards

- Follow the existing code style in the repository
- Write meaningful variable and function names
- Include appropriate comments for complex logic
- Ensure your code is tested before submission

## Commit Messages

Use clear and descriptive commit messages that explain the **why** behind your changes:

- Use imperative mood ("Add feature" not "Added feature")
- Keep the first line under 50 characters
- Reference relevant issues when applicable
- Include a more detailed explanation if needed

Example:
```
Add user authentication endpoint

This implements OAuth 2.0 authentication for secure user login.
Resolves #123
```

## Pull Requests

When submitting a PR:

- Provide a clear title and description
- Link to related issues
- Ensure all tests pass
- Keep PRs focused on a single feature or fix
- Be open to feedback and iterate as needed

## Code Review Process

All submissions require at least one approval before merging. During review:

- Be respectful and constructive
- Ask clarifying questions if needed
- Provide actionable feedback
- Approve once you're satisfied with the changes

## Reporting Issues

When reporting bugs:

- Use a clear, descriptive title
- Include steps to reproduce
- Provide expected and actual behavior
- Include relevant environment information
- Add screenshots if applicable

Thank you for contributing to our project!
